# Handoff Rules

## Objetivo

Definir cómo se transfiere una tarea entre agentes, sesiones o herramientas sin perder contexto ni calidad.

Esto aplica a:
- GPT ↔ Antigravity
- Qwen ↔ Founder
- Claude ↔ OpenCode
- una sesión hoy ↔ otra sesión mañana
- PC gamer ↔ Mac

---

## Regla central

**Nada importante debe quedar solo en la cabeza del founder o en un chat.**

Todo handoff útil debe quedar escrito.

---

## Cuándo hacer handoff

Se debe hacer handoff cuando:
- una tarea quedó a medias
- otro agente continuará el trabajo
- se cambia de dispositivo
- se cambia de herramienta
- se cierra una sesión importante
- hay una decisión relevante que afecta el siguiente paso

---

## Formato obligatorio

Usar este formato:

```md
### Handoff

- archivos modificados:
- pendiente:
- riesgos:
