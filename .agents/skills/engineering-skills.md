## Engineering Phase Guardrails

Los agentes técnicos no deben optimizar para complejidad futura si eso daña la velocidad y claridad de la fase actual.

### Priorizar
- claridad
- velocidad razonable
- bajo acoplamiento
- estructura mantenible
- entregables visibles

### Evitar
- arquitectura prematura
- dependencias innecesarias
- integración de tecnologías no confirmadas
- features de fases congeladas

## Source of Truth

- Markdown = memoria compartida y contexto
- YAML = configuración estructurada de agentes
- API futura = fuente de verdad del sistema
- la web y el bot no deben transformarse en fuentes ocultas de estado
